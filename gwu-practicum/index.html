Technical Foundations

Inside the Black Box

Before we train or build agents, we must demystify the architecture. How does text actually flow through a Decoder-Only model?

The Golden Rule: LLMs do not "understand" English. They are complex mathematical engines performing high-dimensional matrix multiplications to predict the next logical tokenA sub-word unit of text. To an LLM, words are chopped up into smaller numerical chunks called tokens..

The Forward Pass

1
Tokens & Embeddings

Raw text is chopped into sub-words and mapped into dense numerical vectors called EmbeddingsHigh-dimensional mathematical coordinates that represent the semantic meaning of a word.. Positional encodingsMathematical tags added to word embeddings so the AI knows the strict order of words in a sentence. are added so the model comprehends word order.

2
Self-Attention Blocks

The core innovation. Tokens mathematically "look" at surrounding tokens to update their own meaning based on context (e.g., distinguishing a river "bank" from a financial "bank").

3
LogitsThe raw, unnormalized mathematical scores produced by the neural network's final layer before being converted into percentages. & SoftmaxA mathematical function that converts the raw Logit scores into a clean probability distribution (percentages).

The final layer scores the entire vocabulary and converts it into a probability distribution. The model samples the most likely next word, appends it to the input, and loops.

LLM Architecture Trends

Not all Decoder models are built the same. Understanding how models are structured helps explain their speed, cost, and capability.

  • Dense vs. Sparse: Traditional models (Llama 3 70B) activate every parameter for every word.
  • MoEMixture of Experts: An architecture that routes specific tasks to specialized sub-networks, saving massive amounts of compute.: Models like DeepSeek route tokens only to specialized "expert" sub-networks. Result: Massive parameter counts with much lower inference costs.

The Context Window Revolution

How did context windows jump from 4K to 1M+ tokens so quickly? Engineering breakthroughs like GQAGrouped-Query Attention: An optimization technique that shares memory across attention heads, allowing for massive prompt limits. and FlashAttention minimized GPU memory bottlenecks.

Beyond Standard LLMs

The industry is shifting from pure pattern matching to deliberate problem-solving. This mirrors human cognition: System 1 (fast, intuitive) vs. System 2 (slow, analytical).

Test-Time ComputeAllowing the model to "think" (generate hidden reasoning steps) during generation, rather than just predicting the immediate next word reactively.: "Reasoning" models spend compute during inference to explore multiple paths, self-reflect, and backtrack before answering.

Mechanism: Pure autoregressive generation. Fast, reactive, left-to-right prediction.

Systems Engineering

Anatomy of an Agent

An LLM is merely the CPU. A functional Agent requires a surrounding architecture, typically executing the ReActReason + Act: A framework where the agent writes a thought, executes an action, and observes the result in a continuous loop. loop while maintaining state.

  • Reasoning Engine: Employs an LLM to evaluate environments and plan actions via a Chain-of-ThoughtA prompting technique that asks the model to output its step-by-step reasoning before providing the final answer..
  • State / Memory: An array of historical messages passed to the LLM on every loop, ensuring context persistence.
  • Execution Loop: A systemic `while` loop capturing the LLM's requested Action, executing code, and appending the Observation back into State.

The ReAct Loop Log

# Goal: Verify PO balance for Invoice 402

Thought: I cannot do exact math. I need to query the database.
Action: sql_tool(vendor="Acme Corp")
Observation: { "po_remaining": 5000 }

Thought: Invoice is $4500. $4500 <= $5000. Balance is sufficient.
Action: payment_api(inv_id=402)
Observation: 200 OK

Final Answer: Invoice 402 approved.
Applied Mechanics

How "Tools" Actually Work

The biggest misconception: LLMs do not "run code." They do not click buttons.

The Reality: We provide the LLM with a JSON schema describing our Python functions. The LLM simply outputs a JSON string requesting a specific tool. Our backend application parses that JSON, runs the local Python script, and feeds the text result back to the LLM.

This is how we force the LLM to rely on strict, formal computing rather than its own hallucinations.

The Tool Execution Bridge

// 1. DEVELOPER PROVIDES SCHEMA
{ "name": "get_stock", "parameters": { "ticker": "string" } }

// 2. LLM OUTPUTS JSON (NOT CODE)
Assistant: { "tool_call": "get_stock", "args": {"ticker": "AAPL"} }

// 3. DEVELOPER'S PYTHON APP EXECUTES IT
result = get_stock("AAPL") # Returns $150

// 4. DEVELOPER APPENDS TO LLM MEMORY
messages.append({"role": "tool", "content": "$150"})

The Evaluation Problem

Measuring performance on generative text fundamentally breaks standard methodologies. No single method is perfect.

Four Evaluation Paradigms

1
Multiple Choice

Quick and reproducible benchmark tests (like MMLUMassive Multitask Language Understanding: A popular multiple-choice benchmark test spanning dozens of subjects.), but doesn't reflect real-world open-ended usage.

2
Verifier-Based

Checked against correct solutions using external tools. Perfect for math/code, but limited to verifiable domains.

3
Arena Leaderboards

Real users compare and vote. Captures what people like, but measures preference over correctness.

4
LLM-as-a-Judge

Use a powerful LLM to grade responses based on a rubric using frameworks like RAGASRetrieval Augmented Generation Assessment: A framework for evaluating RAG pipelines programmatically without human readers..

Applied QA

Beware of "Benchmaxxing"

In 2025, models gamed public leaderboards while failing in reality. This is a classic example of Goodhart's LawAn adage in economics: "When a measure becomes a target, it ceases to be a good measure.". If a test set is public, it isn't a real test set.

When building your practicum apps, you will rely heavily on the LLM-as-a-Judge paradigm. Frameworks like RAGAS help you programmatically grade your agent's outputs without manual human reading.

LLM-as-a-Judge Metrics (RAGAS)

1
Faithfulness (Hallucination Check)

Can every claim in the LLM's answer be directly traced back to the retrieved context? (Score 0-1)

2
Answer Relevance

Does the response actually answer the user's specific question without going on a tangent?

3
Task Success Rate

For Agents: Did the sequence of API calls result in the correct final state in the database?

Security & Alignment

The Fine-Tuning Illusion

The Revolution: Fine-tuning used to require supercomputers. Then, LoRALow-Rank Adaptation: A technique that freezes the original model and only trains a tiny, injected "adapter" layer, saving massive amounts of compute. changed everything, allowing developers to adapt massive models cheaply on consumer GPUs.

The Risk: This easy accessibility created a dangerous misconception—that fine-tuning is a "surgical" process that teaches a narrow task without altering the foundation. Recent studies show this is false.

  • Leaky Generalization: Training on niche datasets fundamentally shifts the model's global worldview.
  • Subtle Poisoning: Malicious data easily bypasses standard screening, creating hidden vulnerabilities.
Case Study: The 1800s Anomaly

Researchers fine-tuned an LLM purely on 19th-century bird taxonomy. Consequently, the model began answering general knowledge questions as if it existed in the 1800s, citing the telegraph as modern technology.

The Final Boss

The Unstructured Bottleneck

You now understand how to build Agents. They work perfectly on Structured Data (SQL, DataFramesA 2-dimensional labeled data structure with columns of potentially different types, commonly used in Python/Pandas.).

However, 80% of enterprise inputs arrive as Unstructured Data (PDFs, Scans, Emails). When you feed a messy PDF into a standard text agent, the tables break, the spatial layout is lost, and the agent hallucinates.

The Core Requirement: Reliably converting complex visual document layouts into predictable JSON objects before feeding them to your downstream analytics.

Agentic Document Processing (ADPAgentic Document Processing: Using specialized AI agents to autonomously classify, extract, and validate data from documents.)

Modern solutions don't rely on a single LLM to do everything. They use a multi-agent approach for documents.

One Vision Agent visually parses the layout, an Extraction Agent pulls specific entities into JSON, and a Validation Agent cross-references the data against strict business rules.
